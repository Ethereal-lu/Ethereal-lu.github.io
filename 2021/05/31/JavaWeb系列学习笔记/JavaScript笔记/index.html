javaScript 也是万物皆对象。

参数处理

调用函数时，若传入的参数比需要的参数少，或者参数的类型不正确，js会返回NaN；若传入的参数比需要的多，函数只接受前n个他需要的参数，其余参数不理会，但是也不报错。故需要我们手动处理参数。

关键字 arguments 是一个数组，其中存储了调用函数时传入的全部的参数。

关键字 **rest **是一个数组，其中存储了调用函数时传入的多余的参数。

变量作用域

let和var的区别以及const， 一句话就是 var 为函数作用域， let 为块作用域

和java一样，变量选择遵守就近原则，即从自己所在的层级开始查找，如果没有，向上一级查找，直至找到window，若还是没有就报错undefined

所有的全局变量都是 window 对象的属性，只要是定义在函数外的变量就是全局变量，函数本身也属于全局变量。

问题：由于所有的全局变量都会注册为window的属性，则当多个js文件的全局变量都注册到window时，若发生重名，就会产生冲突。

解决方法：每个用户将自己的全局变量都注册到自己的命名空间中，仅将自己的命名空间注册到window。

操作 BOM 对象 【重点】

BOM (Browser Object Model) 是指浏览器对象模型，是用于描述一种对象与对象之间层次关系的模型。

window 对象代表浏览器本身

navigator 封装了浏览器的信息。（不建议使用这个，因为这些信息可以被人为修改）

screen 封装了屏幕的信息。

location 表示当前页面的 url 信息

document 代表当前页面 【最重要的功能即为操作 DOM 树】

history 代表浏览器的历史记录

操作 DOM 对象 【重点】

DOM（Document Object Model）指 文档对象模型。

浏览器页面就是一个 DOM 树形结构！
